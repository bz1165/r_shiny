alt_counts_df — column denominator control

Purpose
-------
Use alt_counts_df when the tabulation dataset is NOT the correct denominator source.

Classic example
---------------
AE SOC/PT table:
- main dataset = subject-level AE SOC/PT records
- denominator should still be SAF subject counts from ADSL

Why this matters
----------------
If you let rtables use the AE row dataset itself for column counts,
the denominator can be wrong because AE datasets contain repeated records.

Typical pattern
---------------
tbl <- build_table(
  lyt,
  ae_socpt_subj,
  alt_counts_df = ex_adsl
)

Mental model
------------
- lyt + ae_socpt_subj control the visible rows and cell values
- alt_counts_df controls .N_col and column denominators
